# 《废土花开》开发任务清单 - 阶段四 (Polish & Early Access Prep)

- [ ] **任务 31：UI 视觉打磨与高压氛围提示**
  - 完善物品栏、HUD、天赋/状态面板的废土风格 UI。
  - 夜幕降临时触发醒目警告UI（“夜幕降临，辐射浓度即将上升”）及急促心跳音效。

- [ ] **任务 32：动态环境音效与音乐系统**
  - 导入背景音乐：庇护所治愈音、废土白天荒凉音、夜晚紧张音乐。
  - 实现基于区域与时间的平滑混音过渡机制：建议使用 `AudioStreamPlayer` 配合 `AnimationPlayer` 控制音量曲线，实现“白天→黄昏→夜晚”的三段无缝过渡。

- [ ] **任务 33：动作与交互音效完善**
  - 补充角色与怪物的脚步声（区分不同地块材质）。
  - 完善种田动作、武器受击反馈、怪物死亡等核心音效。

- [ ] **任务 34：游玩模式分离 (田园 vs 标准)**
  - 新建存档界面提供模式选择。
  - 田园模式：野外怪物不攻城，辐射只减速不致死；标准模式维持常规高压难度。

- [ ] **任务 35：碎片化叙事与图鉴收集系统 (MVP 版)**
  - 在固定废弃场景放置少量“旧时代日记”或“未污染的八音盒”。
  - 制作简易图鉴 UI，收集物品后提供极微小的被动属性奖励。

- [ ] **任务 36：存档与读档系统基础 (重点兼容性验证)**
  - 实现基于 JSON 或 ConfigFile 的序列化保存。
  - 记录并恢复玩家属性、背包物品、时间天数与 TileMap 改造状态（必须使用 `TileMap.get_used_cells()` 序列化以确保净化翻转后读档不会错乱，此机制建议在第三阶段净化功能完成后立刻验证）。

- [ ] **任务 37：主菜单与过渡界面**
  - 制作具有反差美学风格的游戏主菜单。
  - 添加场景与昼夜切换时的黑屏淡入淡出 (Fade) 过渡效果。

- [ ] **任务 38：性能优化验证**
  - 确保视锥外的植物使用 Data 驱动生成。
  - 验证废土粒子特效的 VisibleOnScreenNotifier2D 剔除逻辑，确保帧率稳定。

- [ ] **任务 39：数值平衡与手感最终调优**
  - 平衡体力消耗速度、植物生长周期与打怪收益。
  - 调整紧急净化炸弹与高能纯水的获取概率，确保策略博弈的爽快感。

- [ ] **任务 40：自动化测试与 Demo 导出**
  - 编写自动化测试脚本（使用 Godot 的 SceneRunner 等工具），自动模拟并跑通 7 天流程，确保核心状态机与场景流转不卡死。
  - 进行从开局到第 7 天 Boss 战及净化演出的全流程人工验证，修复恶性穿模等问题。
  - 导出 Windows/macOS 可执行文件及 HTML5 体验版。
